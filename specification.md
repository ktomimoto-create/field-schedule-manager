# フィールド行動予定管理システム 仕様書 (specification.md)

本書は、フィールド行動予定管理システムにおけるスタッフ管理、同期処理、および各画面での表示に関する仕様を定義します。

---

## 1. スタッフの分類とマスタ仕様

システムに登録されるスタッフは、大きく**「FTS社員」**と**「外注メンバー（協力会社社員）」**に分類されます。

### 1.1 FTS社員
* **特徴**: 社員番号（`employee_code`）および本物のメールアドレス（`@fts.co.jp` 等）を持ちます。
* **アバター解決**: Microsoft Entra ID (Azure AD) のプロファイルと連携し、社員番号またはメールアドレスをキーとして自動的にプロフィール画像（アバター）を取得します。
* **自動同期**: 「Microsoftアカウント同期」機能の対象となり、未設定のメールアドレスや社員番号が自動で補完されます。

### 1.2 外注メンバー（マスタ外対応者）
* **特徴**: 協力会社のメンバーであり、FTSの社員とは異なります。
* **判定基準**: 
  - 氏名が `FE`, `SF`, `FR` プレフィックスから始まる（例: `SF濱田`、`FR岡崎`）
  - または、デフォルトコース番号が `90` 以上
* **メールアドレス・社員番号**: 
  - メールアドレスはダミー（例: `hamada@example.com`）を原則保持します。
  - 社員番号 (`employee_code`) は `null` (未設定) とします。
* **アバター解決**: Microsoft アカウントとの紐付けを行わないため、画面上は名前の頭文字を用いた**「イニシャルバッジ」**（例: `S` や `F`）で表現されます。
* **自動同期**: **同期処理から完全に除外（自動スキップ）**されます。名前の部分一致などによって、同姓のFTS社員（例: 濱田正貴、岡﨑勝二）と誤って紐付けられることを防ぎます。

## 2. 自動同期仕様 (Microsoftアカウント同期)

マスタ管理画面の「Microsoftアカウント同期」を実行した際の同期仕様は以下の通りです。

1. **除外ガード**:
   - 対象スタッフが外注メンバー（`FE`, `SF`, `FR` で始まる、またはコース番号90以上）である場合、同期処理を自動的にスキップします。
2. **既存アドレスの保護**:
   - すでにダミーアドレス（`@example.com`）以外の本物のメールアドレスが登録されているスタッフは、上書きを防ぐため同期処理をスキップします。
3. **名寄せキー優先順位**:
   - **社員番号による完全一致**: スタッフに社員番号が登録されている場合、外部プロファイルの `employee_id` との完全一致で紐付けます（最優先）。
   - **名前による一致**: 社員番号が未登録、または一致しない場合、スタッフ名からプレフィックスを除去したクリーンな苗字と、外部プロファイルの `display_name`（スペース除去・表記揺れ正規化後）を突合します。
     - ただし、同姓同名の被りを防ぐため、完全一致するプロフィールが「唯一（1件のみ）」存在する場合のみマッチと判定します。複数マッチした場合は警告を出力してスキップします。
     - ※ 「フーギー（ナルマンダフ・フスレンバヤル）」などの特殊名については個別の例外マッチングルールを適用します。
4. **同期内容**:
   - マッチした外部プロファイルのメールアドレス（小文字化）および社員番号（未設定の場合のみ）を、Supabaseの `staff` テーブルに書き込みます。

## 3. UI表示およびソート仕様

### 3.1 マスタ管理画面 (MasterManagementView)
* **スタッフ一覧**:
  - 氏名列には、**アバター画像（またはイニシャルバッジ） ＋ 登録氏名（プレフィックスを含むフルテキスト、例: `SF濱田`）**を表示します。
  - 外注メンバーなどアバター画像がないスタッフは、登録氏名の頭文字を表示する丸型バッジとなります。
  - FTS社員などを表示する際、画面上で名前を省略せず、マスタに登録されている氏名でフル表示します。

### 3.2 当日行動予定表 (TimelineView)
* **スタッフボードのソート順**:
  - 画面に表示されるスタッフボード（各スタッフごとの列）は、デフォルトコース（`default_course`）の**数値昇順**（例: 1, 2, ..., 26, 27, 90, 91...）で左から右へ並び替えて表示します。
  - コース番号が数値に変換できない、または未設定のスタッフは、リストの末尾に配置されます。

### 3.3 その他の画面での表示形式（既定）
* **当日・月間行動予定表**: アイコン ＋ 苗字（getShortName による変換後）
* **行動予定**: 苗字のみ

## 4. 月間予定表（CalendarView）における予定ステータス仕様

### 4.1 予定のステータスとデザイン表現
すべての予定セルは視認性を考慮して**白背景（通常背景）**を基本とし、手動で割り振られたステータスに応じて以下のアクセントを適用します。

* **通常（フリー）予定 (free)**:
  - 新規作成時や、ステータスが手動割り当てされていない状態の標準予定です。
  - 背景色：白（無色のセル）
  - 左端のアクセントバー：なし
* **確定予定 (confirmed)**:
  - 背景色：白
  - 行の先頭セル（`type` 列のセル）の左端に、太さ 4px の **グリーンのアクセントバー (#10b981)** を表示して確定状態を示します。
* **仮予定 (draft)**:
  - 背景色：白
  - 行の先頭セル（`type` 列のセル）の左端に、太さ 4px の **オレンジのアクセントバー (#f59e0b)** を表示して仮の状態を示します。
* **キャンセル (cancelled)**:
  - 背景色：灰色 / 取消線

### 4.2 ステータスのワンクリック切り替え
* 予定セルを右クリックした際に表示されるカスタムコンテキストメニューから、現在のステータス以外のトグル（例: **「予定を【フリー】に変更」**、**「予定を【確定】に変更」**、**「予定を【仮】に変更」**）をワンクリックで実行できます。
* 選択したステータスは即座にデータベース（Supabase）に反映され、再読み込みなしで画面上が更新されます。
